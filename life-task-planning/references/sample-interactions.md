# Sample Interactions - Life & Work Task Planning

This file contains detailed conversation examples showing how the life-task-planning skill should interact with users in different scenarios.

## Scenario 1: First-Time User (Empty Backlog)

**User:** "更新"

**Claude:**
好的！我可以帮你管理个人和工作任务。看起来你还没有任务列表，让我们开始创建吧。

你目前有哪些事情需要追踪？比如工作项目、个人目标、待办事项等。

---

**User:** "我需要完成季度报告，还有这周要去看牙医"

**Claude:**
好的，我来帮你添加这两个任务。让我先了解一下：

**关于"完成季度报告"：**
- 这个任务的紧急程度如何？（高/中/低）
- 预计何时需要完成？

---

**User:** "高优先级，本周五前完成"

**Claude:**
明白了。

**关于"看牙医"：**
- 已经预约了具体时间吗？
- 这个任务的紧急程度如何？

---

**User:** "周三下午2点，普通优先级就行"

**Claude:**
好的！让我为你添加这两个任务：

1. **完成季度报告** - 高优先级，本周五前
2. **看牙医** - 中优先级，周三下午2点

这样安排可以吗？

---

首先，你目前手头有哪些正在进行或计划中的事情？还是从这些现有任务开始？

---

**User:** "季度报告已经完成一半了"

**Claude:**
太好了！已经完成一半了，进展不错。

关于季度报告，有什么变化吗？需要调整优先级或截止时间吗？

---

**Claude:**
好的，我来更新一下季度报告的进度。

那"学习 Python"呢？进展如何？

---

## Scenario 3: Quick Add Mode

**User:** "新任务：准备下周的演示文稿"

**Claude:**
好的！让我了解一下这个任务：

1. **紧急程度如何？**（高/中/低）
2. **预计何时需要完成？**

---

**User:** "高优先级，下周三需要"

**Claude:**
我看你本周已经有 2 个高优先级任务了。这个新任务可以：

- **Option A:** 作为本周的重点开始准备
- **Option B:** 周末完成季度报告后再集中处理
- **Option C:** 暂时记下来，下周再详细规划

你觉得哪个方案比较合适？

---

**User:** "选 B"

**Claude:**
好的，已添加任务："准备下周的演示文稿" - 高优先级，安排在季度报告之后。

还有其他需要调整的吗？

## Scenario 4: User Feeling Overwhelmed

**User:** "更新... 哎，这周事情太多了"

**Claude:**
听起来这周确实压力挺大的 😟

让我看看你现在的任务：
1. [ ] 完成季度报告 - 高优先级，本周五
2. [ ] 准备演示文稿 - 高优先级，下周三
3. [ ] 学习 Python - 中优先级
4. [ ] 整理书房 - 低优先级
5. [ ] 看牙医 - 中优先级，周三下午

这周确实安排得很满。要不我们先聚焦在最重要的 2-3 个任务上，其他的可以暂时推后？

你觉得哪些任务这周必须完成，哪些可以放一放？

---

**User:** "季度报告必须完成，牙医不能改。其他都可以推后"

**Claude:**
明白了，这样就清晰多了。让我帮你调整一下：

**本周必做：**
1. 完成季度报告（周五前）
2. 看牙医（周三下午）

**暂时推后：**
- 准备演示文稿 → 下周再处理
- 学习 Python → 暂缓，等忙完这段时间
- 整理书房 → 等有空再安排

这样安排感觉轻松些了吗？

## Scenario 6: Handling Dependencies

**User:** "新任务：发布产品更新"

**Claude:**
好的！让我了解一下：

1. **紧急程度如何？**（高/中/低）
2. **预计何时需要完成？**
3. **这个任务依赖其他事项吗？** 比如需要先完成测试、文档等？

---

**User:** "高优先级，下周一发布。需要先完成测试和文档"

**Claude:**
明白了。我看到你已经有"准备演示文稿"在进行中。

让我这样安排：
- 新建任务："发布产品更新" - 高优先级，下周一
- 添加子任务：完成测试、撰写文档
- 设置依赖：依赖任务完成后才能发布

这样可以吗？

## Key Interaction Patterns

### Pattern 1: One Question at a Time
❌ Bad: "紧急程度是什么？什么时候完成？有什么依赖？谁负责？"

✅ Good: 先问紧急程度，等待回答，再问下一个问题

### Pattern 2: Show Before Changing
❌ Bad: 直接修改任务列表

✅ Good: "我计划这样调整...你觉得可以吗？"

### Pattern 3: Empathy First
❌ Bad: "你任务太多了，删除一些"

✅ Good: "听起来这周确实挺忙的。要不我们先聚焦最重要的？"

### Pattern 4: Concrete Suggestions
❌ Bad: "你想要怎么安排？"

✅ Good: "我建议三个方案：A / B / C，你倾向哪个？"

### Pattern 5: Celebrate Wins
❌ Bad: "任务完成。下一个？"

✅ Good: "太棒了！🎉 这个任务不容易。想继续还是先休息？"
